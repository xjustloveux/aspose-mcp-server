✨ 功能特性

了解 Aspose MCP Server 的強大功能

🎯 核心特性

核心功能

  • 115 個統一工具 - Word(27)、Excel(31)、PowerPoint(25)、PDF(19)、OCR(2)、Email(6)、BarCode(2)、轉換(2)、Session(1) 已整合
  • 按需啟用 - 只啟用需要的文檔類型，減少資源佔用
  • 跨平台 - Windows、Linux、macOS (Intel + ARM) 原生支援，無需外部圖形庫，單一可執行檔案
  • 開箱即用 - 預編譯版本無需安裝 .NET Runtime
  • 完整讀寫 - 支援從A文檔讀取格式應用到B文檔

技術特性

  • MCP 2025-11-25 規範支援 - 完全符合最新 MCP 協議規範，自動工具註解（readonly/destructive）基於命名約定，完整的 JSON-RPC 2.0 錯誤處理
  • 統一字型設定 - 多個工具支援中英文字型分別設定（fontNameAscii 和 fontNameFarEast 參數）
  • 靈活的授權配置 - 支援總授權或單一組件授權，自動搜尋、環境變數或命令列參數配置，試用模式降級（找不到授權時）
  • 自動工具發現 - 基於命名約定的自動工具註冊系統
  • 安全加固 - 全面的路徑驗證、輸入驗證和錯誤處理

🔒 安全特性

路徑驗證

  • 所有檔案路徑都經過 `SecurityHelper.ValidateFilePath()` 驗證
  • 防止路徑遍歷攻擊（`../`, `..\`）
  • 限制路徑長度（最大260字元）和檔案名稱長度（最大255字元）
  • 驗證路徑中的非法字元

輸入驗證

  • 陣列大小驗證（`SecurityHelper.ValidateArraySize`，最大1000項）
  • 字串長度驗證（`SecurityHelper.ValidateStringLength`，最大10000字元）

錯誤處理

  • 錯誤訊息清理，防止資訊洩露
  • 移除檔案路徑、堆疊追蹤等敏感資訊
  • 生產環境不暴露詳細錯誤資訊

Origin 驗證（DNS 重綁定攻擊防護）

  • HTTP 和 WebSocket 模式預設啟用 Origin 標頭驗證
  • 支援配置允許的來源域名清單
  • 可配置是否允許 localhost 來源（開發環境）
  • 可配置是否要求必須有 Origin 標頭
  • 健康檢查端點（/health, /ready）自動排除驗證

安全限制

最大路徑長度 260 字元
最大檔案名稱長度 255 字元
最大陣列大小 1000 項
最大字串長度 10000 字元

📡 MCP 協議支援

MCP 2025-11-25 規範支援

  • 符合最新 MCP 協議規範（protocolVersion: 2025-11-25）
  • 使用 ModelContextProtocol SDK 0.6.0-preview.1
  • 完整的 JSON-RPC 2.0 錯誤處理

Tool Annotations（工具行為標註）

  • Title - 人類可讀的工具標題
  • ReadOnly - 標示只讀操作（如 get、list）
  • Destructive - 標示可能造成破壞性變更的操作（如 delete）
  • Idempotent - 標示冪等操作（重複調用產生相同結果）
  • OpenWorld = false - 所有工具都是本地檔案操作，不與外部實體交互

結構化輸出（outputSchema）

  • 所有工具回傳結構化 JSON 結果
  • SDK 自動生成 outputSchema（使用 oneOf 支援多種結果類型）
  • 包含 output 欄位（path 或 sessionId）指示輸出位置
  • 客戶端可根據 schema 自動解析結果

🔍 OCR 文字辨識

文字辨識功能

  • 圖片文字辨識 - 支援 PNG、JPG、BMP、TIFF 等圖片格式的文字辨識
  • 掃描 PDF 轉換 - 將掃描 PDF 轉換為可編輯文件（DOCX、XLSX、PDF、TXT）
  • 多語言支援 - 英文、中文、德文、法文、西班牙文、義大利文、俄文、印地文等
  • 文字定位（Bounding Box） - 使用 includeWords=true 取得每個辨識單詞的座標位置（X, Y, Width, Height），適用於文字位置分析與版面重建

平台支援

  • Windows x64、Linux x64、macOS x64 完整支援
  • macOS ARM64 透過 Rosetta 2 支援
  • Linux ARM64 不支援（ONNX Runtime 限制）

📧 Email 郵件處理

郵件格式轉換

  • MSG 轉 EML - 將 Outlook MSG 格式轉換為標準 EML 格式
  • EML 轉 MSG - 將 EML 格式轉換為 Outlook MSG 格式
  • 完整保留郵件內容、附件與格式

郵件資訊提取

  • 郵件屬性 - 提取主題、寄件者、收件者、日期等資訊
  • 附件列表 - 獲取郵件中所有附件的清單
  • 附件提取 - 將郵件附件提取到指定目錄
  • 附件刪除 - 從郵件中移除指定附件

支援格式

  • MSG - Microsoft Outlook 郵件格式
  • EML - 標準電子郵件格式（RFC 822）

📊 BarCode 條碼處理

條碼生成

  • 1D 條碼 - Code128、Code39、EAN-13、UPC-A 等
  • 2D 條碼 - QR Code、Data Matrix、PDF417、Aztec 等
  • 郵政條碼 - USPS OneCode、Australia Post、Singapore Post 等
  • 自訂樣式 - 尺寸、顏色、解析度等參數可調

條碼辨識

  • 多格式識別 - 自動識別圖片中的條碼類型
  • 批量識別 - 一張圖片中識別多個條碼
  • 支援格式 - PNG、JPG、BMP、TIFF、GIF 等圖片格式

支援的條碼類型（60+種）

  • 線性條碼 - Code128、Code39、Code93、Codabar、ITF-14、ISBN、ISSN
  • 二維條碼 - QR Code、Micro QR、Data Matrix、PDF417、Aztec
  • 零售條碼 - EAN-8、EAN-13、UPC-A、UPC-E
  • 郵政條碼 - Postnet、Planet、Australia Post、Deutsche Post

🎨 其他特性

統一字型設定

多個工具支援中英文字型分別設定（`fontNameAscii` 和 `fontNameFarEast` 參數），確保中英文混排文檔的完美顯示。

靈活的授權配置

  • 支援總授權或單一組件授權
  • 自動搜尋、環境變數或命令列參數配置
  • 試用模式降級（找不到授權時）

跨格式轉換

  • 啟用任何文檔工具時，自動包含 `convert_to_pdf`（轉換為PDF）
  • 啟用兩個或以上文檔工具時，自動包含 `convert_document`（跨格式轉換，如Word轉Excel）

基於 .NET 8.0、Aspose.Total 23.10.0 和 MCP SDK 0.6.0 構建
