🚀 快速開始

幾分鐘內開始使用 Aspose MCP Server

步驟 1：下載預編譯版本

1 從 GitHub Releases 下載

訪問 GitHub Releases 下載對應平台的版本：

  • Windows: aspose-mcp-server-windows-x64.zip
  • Linux: aspose-mcp-server-linux-x64.tar.gz
  • macOS Intel: aspose-mcp-server-macos-x64.tar.gz
  • macOS ARM: aspose-mcp-server-macos-arm64.tar.gz
2 解壓到任意目錄

解壓縮下載的檔案到任意目錄，例如：

  • Windows: C:\Tools\aspose-mcp-server\
  • macOS/Linux: ~/tools/aspose-mcp-server/

macOS 首次執行：需移除系統隔離標記（僅需一次）：

xattr -d com.apple.quarantine AsposeMcpServer
* macOS 替代方案：使用 Homebrew 安裝（推薦） 
brew install xjustloveux/tap/aspose-mcp-server

透過 Homebrew 安裝無需手動處理權限和隔離標記。

透過 Claude Desktop Extension (.mcpb) 安裝

Claude Desktop 使用者可透過 .mcpb 擴充功能套件一鍵安裝，伺服器將以原生方式在您的主機作業系統上執行，不受 Claude Desktop Linux 沙箱限制（Windows 路徑如 C:\Users\... 可直接使用）。

1 下載對應平台的 .mcpb 檔案

從 GitHub Releases 下載：

  • Windows x64: aspose-mcp-server-windows-x64.mcpb
  • macOS ARM64: aspose-mcp-server-macos-arm64.mcpb
  • macOS x64: aspose-mcp-server-macos-x64.mcpb
  • Linux x64: aspose-mcp-server-linux-x64.mcpb
2 雙擊開啟 — Claude Desktop 安裝對話框自動彈出

Claude Desktop 會顯示伺服器名稱、說明與圖示，並提示您選擇授權檔案。

3 選擇 Aspose 授權檔案（或略過以評估模式執行）

透過檔案選擇器指定 Aspose.Total.lic 或單一元件 .lic 檔案。略過時伺服器以評估模式執行，輸出文件將帶有 Aspose 浮水印。

💡 macOS 使用者注意

透過 .mcpb 安裝時，Claude Desktop 會自動處理 macOS 隔離標記，無需手動執行 xattr -d com.apple.quarantine 指令。若您使用 .tar.gz 手動安裝，仍需執行一次 xattr。

🔌 非 Claude Desktop 客戶端

Cursor、Continue 或自訂 MCP 客戶端請使用上方「步驟 1：下載預編譯版本」的 .zip / .tar.gz 方式安裝，並依「步驟 3：配置 MCP 客戶端」手動設定 config.json。

步驟 2：配置授權檔案

📄 源代碼授權

本專案的源代碼採用 MIT License 授權，您可以自由使用、修改和分發源代碼。

📋 Aspose 授權檔案類型

⚠️ 注意：使用本專案需要有效的 Aspose 授權檔案才能正常運行。

  • 總授權：Aspose.Total.lic（包含所有組件）
  • 單一組件授權：Aspose.Words.lic、Aspose.Cells.lic、Aspose.Slides.lic、Aspose.Pdf.lic
  • 自訂檔案名稱：可透過環境變數或命令列參數指定

配置方式（三選一）

1 自動搜尋（推薦）

將授權檔案放在可執行檔案同一目錄，系統會自動搜尋。

2 環境變數

設定 ASPOSE_LICENSE_PATH 環境變數指向授權檔案路徑。

3 命令列參數

使用 --license:路徑 或 --license=路徑 指定授權檔案。

⚠️ 注意：如果找不到授權檔案，系統會以試用模式運行（會有試用版標記）。

步驟 3：配置 MCP 客戶端

1 編輯配置檔案

根據您使用的 MCP 客戶端，編輯對應的配置檔案。配置檔案通常位於應用程式的設定目錄中，請參考您使用的客戶端文檔以確認具體路徑。

2 添加配置

在配置檔案中添加以下內容：

{ "mcpServers": { "aspose-word": { "command": "C:/Tools/aspose-mcp-server/AsposeMcpServer.exe", "args": ["--word"] } } }

可用參數

不帶任何工具參數時，預設啟用所有工具。指定工具參數時，只啟用指定的工具：

  • --word - Word 工具（自動包含轉換功能）
  • --excel - Excel 工具（自動包含轉換功能）
  • --powerpoint / --ppt - PowerPoint 工具（自動包含轉換功能）
  • --pdf - PDF 工具
  • --ocr - OCR 文字辨識工具
  • --email - Email 工具（郵件、日曆、聯絡人）
  • --barcode - BarCode 工具（條碼產生與辨識）
  • --all - 所有工具（等同不帶工具參數）
  • --session-enabled - 啟用 Session 管理功能（可選）
  • --license:路徑 或 --license=路徑 - 指定授權檔案路徑（可選）
🔧 工具過濾機制

MCP 工具列表會根據您啟用的參數動態調整。例如只啟用 --word 時，只會顯示 Word 相關工具，不會出現 Excel、PDF 等其他工具。

傳輸模式

支援三種傳輸模式，可透過命令列參數或環境變數切換：

模式 命令範例 端點 適用場景
Stdio（預設） AsposeMcpServer.exe --word - 本地 MCP 客戶端
HTTP AsposeMcpServer.exe --http --port 3000 --word http://localhost:3000/mcp 網頁應用
WebSocket AsposeMcpServer.exe --ws --port 3000 --word ws://localhost:3000/mcp 即時互動

環境變數配置（Docker/Kubernetes 適用）

除了命令列參數外，也可以使用環境變數配置：

  • ASPOSE_TOOLS - 啟用的工具：all（全部）或 word,excel,pdf,ppt,ocr,email,barcode
  • ASPOSE_TRANSPORT - 傳輸模式：stdio、http、ws
  • ASPOSE_PORT - 監聽埠號（預設：3000）
  • ASPOSE_HOST - 監聯位址（預設：localhost，容器用 0.0.0.0）
  • ASPOSE_SESSION_ENABLED - 啟用 Session 管理：true 或 false
  • ASPOSE_SESSION_ISOLATION - Session 隔離模式：none（無隔離）、group（群組級，預設）
  • ASPOSE_SESSION_AUTO_SAVE_INTERVAL - 自動暫存間隔（分鐘，0 = 停用，預設：0）
💡 轉換功能說明
  • 啟用任何文檔工具（--word、--excel、--ppt、--pdf）時，自動包含 convert_document（跨格式轉換，支援 Word/Excel/PDF 轉圖片、特殊格式轉 PDF）

步驟 4：重啟 MCP 客戶端

完成配置後，重啟您使用的 MCP 客戶端即可開始使用。

✅ 完成！ 現在您可以在 MCP 客戶端中使用 Aspose MCP Server 的所有工具了。

📝 使用範例

從A文檔複製格式到B文檔

複製段落格式：

1. word_paragraph(path="A.docx", operation="get_format", paragraphIndex=0) 2. 使用返回的格式資訊 3. word_paragraph(path="B.docx", operation="edit", paragraphIndex=0, ...)

複製表格結構：

1. word_table(path="A.docx", operation="get_structure", tableIndex=0) 2. 參考返回的結構資訊 3. word_table(path="B.docx", operation="create", ...) 創建相同結構

複製樣式：

word_style(path="B.docx", operation="copy_styles", sourceDocument="A.docx")

基於 .NET 8.0、Aspose.Total 23.10.0 和 MCP SDK 1.2.0 構建
